# WORKFLOW — Как устроен наш рабочий процесс

---

## Роли

### Cowork (я — Claude в этом диалоге)
- Пишу **промпты для Claude Code** — подробные, точные, с контекстом
- Даю **пояснения Артёму** — что делается, зачем, что ожидать
- Веду **планирование и верификацию** — проверяю план, закрываю пробелы
- НЕ пишу код напрямую — только инструкции

### Артём (пользователь)
- Копирует промпты в **Claude Code**
- Получает ответ Claude Code и скидывает результат обратно мне
- Принимает решения (approve/reject) на ключевых точках
- Задаёт вопросы когда что-то непонятно

### Claude Code (исполнитель)
- Получает промпты от Артёма
- Имеет доступ к проекту (pipeline/, prompts/, config.yaml, tests/)
- Имеет CLAUDE.md как project instructions
- Создаёт файлы, запускает тесты, делает git операции
- НЕ видит наш с Артёмом диалог — только то что Артём ему пишет

---

## Цикл работы

```
1. Я (Cowork) пишу промпт для Claude Code
      ↓
2. Артём копирует промпт → вставляет в Claude Code
      ↓
3. Claude Code создаёт план → Артём проверяет / спрашивает меня
      ↓
4. Claude Code выполняет → Артём скидывает мне результат
      ↓
5. Я проверяю → даю следующий промпт
      ↓
   Повторяем до завершения фазы
```

---

## Формат промпта

Каждый промпт для Claude Code содержит:
1. **Что создать** — конкретные файлы и их содержимое
2. **Контекст** — зачем это нужно, как связано с планом
3. **Ограничения** — что НЕ делать
4. **Верификация** — как проверить что всё правильно

Промпт оформляется в блоке кода (```) чтобы Артём мог скопировать целиком.

## Артёму на заметку

### Если Claude Code предлагает изменить план
→ Скинь мне его предложение. Я оценю и решим вместе.

### Если Claude Code делает что-то непонятное
→ Скинь мне его ответ. Я объясню.

### Если сессия заканчивается
→ Начни новый диалог со мной (Cowork). Скажи "продолжаем работу". Я прочитаю .context/ и подхвачу.

### Если Claude Code теряет контекст
→ Скинь ему IMPLEMENTATION_PLAN_FINAL.pdf + ARCHITECTURE_TREE.pdf заново. Плюс скажи на какой фазе мы.

## Ключевые файлы проекта

| Файл | Что содержит | Когда читать |

| .context/SKILL_PROMPTS.md | Готовые промпты для скиллов | При создании скиллов |
