Content Stream — Логика

Академические стримы (Курсовая, Реферат, ВКР, Исследование) генерируют по секциям — каждый Worker-агент отвечает за одну секцию/главу. Worker-агент получает контекст предыдущих секций + мегапромт + источники от SourceFinder.

Текст делится на 4 подтипа: аналитический (data-driven), публицистический (opinion/argumentation), обзорный (literature/theme survey), описательный (narrative). За каждым — отдельный Worker со своим регистром и domain.

Эссе и Сочинение генерируются целиком одним Worker-агентом (объём не требует декомпозиции на главы).

Humanizer Stream — Логика

5 Micro Managers (HM-1→HM-5) = 5 стадий pipeline.
Каждый HM содержит 3 специализированных Worker-агента.
HM-5 (Контролёр) имеет Feedback Loop: при fail отправляет текст обратно к нужному HM.
Language Gate — общий сервис, доступный всем HM.

Humanizer — Альтернативная декомпозиция (для обсуждения)

Вариант A (текущий — по стадиям pipeline): HM-1 Диагност → HM-2 Архитектор → HM-3 Лексикограф → HM-4 Голос → HM-5 Контролёр. Плюсы: чёткая последовательность, каждый агент — узкий специалист. Минусы: если fail, надо проходить цепочку заново.

Вариант B (по типу паттернов): HM-Content (P1–P6), HM-Language (P7–P12), HM-Style (P13–P18), HM-Discourse (P25–P30), HM-Corpus (P31–P53), HM-Format (F1–F8). Плюсы: каждый агент отвечает за свою категорию паттернов. Минусы: паттерны пересекаются между стадиями — один агент не может устранить P7 без знания домена из Stage 1.

Вариант C (гибрид — по функциональной роли): HM-Analyst (диагностика), HM-Surgeon (глубокая переработка = Stage 2+3), HM-Soul (голос = Stage 4), HM-QA (скоринг + loop). Плюсы: меньше агентов (4 vs 5), HM-Surgeon объединяет структуру + лексику. Минусы: HM-Surgeon перегружен.

Иерархия агентов — Общие принципы

CEO Agent — маршрутизирует задачи по stream_id. Не генерирует текст. Мониторит финальное качество.
Manager Agent (Content / Humanizer) — координирует Micro Managers. Определяет domain, register, language.
Micro Manager — контролирует Workers в своём стриме. Знает секционный порядок, передаёт контекст между Workers.
Worker Agent — самый ограниченный. Генерирует/обрабатывает одну секцию или один тип преобразования. Имеет свой промт, свой skill, свои constraints.

Правило коммуникации: Worker → Micro Manager → Manager → CEO (только вверх). Горизонтальная коммуникация между Workers одного уровня — через Micro Manager.
