v3 Fixes — Content Stream (Problems #1–#5)

#1 — SourceFinder FIRST: Перемещён с последнего Worker на ПЕРВЫЙ во всех академических MM (Курсовая, Реферат, ВКР, Исследование). Теперь source_list доступен всем Content Workers через контекст MM. Текст/Эссе/Сочинение — не используют SourceFinder.

#2 — TOC Worker 2-pass: Содержание генерируется дважды — placeholder на этапе генерации, финальная версия после сборки всех глав (MM собирает heading list и обновляет TOC).

#3 — Content QA Gate: Новый компонент после всех Workers каждого MM. Проверяет: announcement openers=0, triplets=0, Block 12 violations=0, Block 7 red flags≤2. Fail → MM перегенерирует секцию (max 2 попытки).

#4 — Visualization Tracking: Content QA Gate считает tables+figures в выходе и сверяет с минимумом stream_id (БЛОК 16.2). VKR бак≥5, маг≥8, coursework≥3, research≥3. Fail → re-generate data-heavy секции с инструкцией добавить визуализации.

#5 — Content→Humanizer Handoff: Новый Handoff Gate между стримами. Передаёт: text, domain, register, language, source_list, viz_count, structural_check_report. В humanization mode — CEO направляет текст прямо в Handoff, минуя Content Stream.

v3 Fixes — Humanizer Stream (Problems #6–#11)

#6 — Domain Classifier → Domain Gate sync: HM-1 Domain Classifier теперь ПИШЕТ результат в Domain Gate (Shared Service). Все последующие HM читают domain_profile оттуда. Устраняет дублирование логики и неясность взаимодействия.

#7 — Inter-stage Coherence QA: 3 QA Gate вставлены между стадиями: HM-1→[QA]→HM-2, HM-2→[QA]→HM-3, HM-3→[QA]→HM-4. Используют embedding-based cosine similarity (SentenceTransformer) — если coherence drops ≤ 0.85 после стадии, текст возвращается на повторную обработку.

#8 — Register Guard (QA Gate #3): QA Gate между HM-3→HM-4 дополнительно проверяет register drift. Сравнивает register profile выхода HM-3 с baseline из HM-1. Флаг если formality shift > допустимого порога (конфиг: register_drift_threshold).

#9 — AC-5 Figurative Language Flag: Language Gate теперь хранит flag skip_figurative=true для комбинации language=ru + register=academic. HM-4 Soul Injector проверяет этот flag ПЕРЕД Op 8 (metaphor injection). Пропускает Op 8 полностью если flag=true.

#10 — Pattern Scanner как Shared Service: Вынесен из Workers HM-1 в Shared Service. Используется: (a) HM-1 для baseline scan, (b) HM-5 Score Engine для rescan, (c) Content QA Gate для structural check. Единый движок — согласованные результаты.

#11 — One-transform-per-prompt: Архитектурно подразумевается — каждый Worker делает ровно один тип трансформации через API Gateway. Не требует отдельного компонента.

v3 Fixes — System-Level (Problems #12–#13)

#12 — API Gateway: Новый Shared Service. Единая точка для ВСЕХ Claude API вызовов (Phase 0B generation, Stage 2–4 rewrites). Rate limiter с exponential backoff, token budget tracking, retry logic. Все Workers вызывают API только через Gateway.

#13 — CEO Routing Table: CEO Agent теперь имеет формализованную routing table:
• Generation mode: SourceFinder → Content MM → Content QA → Handoff → HM-1→5 → DOCX Formatter → Output
• Humanization mode: Input → Handoff → HM-1→5 → DOCX Formatter → Output
• Fail escalation: после 2 feedback loops → CEO получает fail report, принимает решение (skip metric / re-generate from scratch / partial accept)

Feedback Loop — Маршрутизация при fail

HM-5 Feedback Router определяет КУДА вернуть:

→ HM-2: fail по para_cv, section_cv, triplets, connector density, scaffold signals (P44–P53)
→ HM-3: fail по P7 violations, modal hedging, attribution, elegant variation
→ HM-4: fail по burstiness, perplexity lift, But:However ratio, imperfection texture, voice absence

Max 2 итерации. После 2-го fail → escalation к CEO Agent с полным score_report.

Общая статистика архитектуры v3

CEO: 1 агент (с routing table)
Manager: 2 агента (Content + Humanizer)
Handoff Gate: 1 агент (Content→Humanizer)
Content Micro Manager: 7 агентов
Content QA Gate: 1 агент (shared across all MMs)
Content Worker: 30 агентов (с перемещённым SourceFinder)
Humanizer Micro Manager: 5 агентов
Humanizer Inter-Stage QA: 3 агента (coherence + register)
Humanizer Worker: 19 агентов (Pattern Scanner → SS)
Shared Service: 7 сервисов (Language Gate, Domain Gate, SourceFinder, DOCX Formatter, Chunk Manager, Pattern Scanner, API Gateway)

ИТОГО: 69 агентов + 7 shared services = 76 компонентов

Все LLM-вызовы проходят через API Gateway.
Pattern Scanner — единый движок для HM-1, HM-5, Content QA.
Chunk Manager — единый для HM-2, HM-3, HM-4.
Coherence QA gates гарантируют отсутствие деградации между стадиями.

Data Flow — End-to-End Trace (VKR пример)

1. CEO получает params: stream_id=vkr, topic=..., domain=economics, language=ru, level=bachelor
2. CEO → Content Manager → MM-3 (ВКР)
3. MM-3 вызывает SourceFinder Service ① → получает source_list (≥50 ГОСТ-формат)
4. MM-3 последовательно: Тит. лист → Аннотация → TOC (placeholder) → Введение → Гл.1 → Гл.2 → Гл.3 → Заключение
5. MM-3 обновляет TOC (2nd pass) с реальными heading + страницами
6. Весь текст → Content QA Gate: structural check + viz count (≥5). При fail → перегенерация секции
7. CEO → Handoff Gate: text + domain=economics + register=academic + language=ru + sources + viz_count
8. HM-1 Диагност: Pattern Scanner (SS) → baseline metrics. Domain Classifier → пишет в Domain Gate: economics profile
9. [QA Coherence] → HM-2 Архитектор: Scaffold Breaker, Format Cleaner, Connector Tuner, Triplet Buster. Chunks via Chunk Manager (SS). API calls via API Gateway (SS)
10. [QA Coherence] → HM-3 Лексикограф: Vocab Eliminator (с domain exemption от Domain Gate), Hedging Auditor, Attribution Fixer, Result Formatter. Language Gate: AC-2 ГОСТ [N] формат, AC-6 Russian patterns
11. [QA Register] → проверка register drift. Если OK → HM-4 Голос: Soul Injector (Language Gate: AC-5 skip figurative), Rhythm Shaper, Pass B Auditor, Example Loader. Language Gate: AC-4 passive 70%, AC-7 skip F5/F6
12. HM-5 Контролёр: Score Engine (Pattern Scanner SS для rescan), Detector Probe (opt), Feedback Router. Fail → routing к HM-2/3/4 (max 2). Pass → Report Builder
13. CEO → DOCX Formatter (SS): ГОСТ 7.32-2017, Times New Roman 14pt, 1.5 spacing, tables+figures
14. Output: .txt + .docx + score_report.json + score_report.txt
